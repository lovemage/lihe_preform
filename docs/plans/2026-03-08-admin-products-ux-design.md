# 2026-03-08 Admin Products UX 改版設計

## 目標

本次改版聚焦於 admin 後台的產品管理體驗，優先提升編輯效率與可視化程度，並統一後台介面語言與版型。

## 範圍

### 1. Admin 整體介面

- 全部 admin 文案改為繁體中文。
- 將目前頂部導覽改為左側固定側欄。
- 側欄保留以下入口：
  - 儀表板
  - 首頁內容
  - 產品管理
  - 媒體庫
- 主內容區維持右側顯示頁面標題與內容。

### 2. Products 列表頁

- 將頁面標題、描述、按鈕與表格欄位名稱改為繁體中文。
- 維持現有產品清單與進入編輯頁的流程。

### 3. Product 編輯頁

採用單頁整合式編輯器（方案 A）：

- 基本資料區
  - slug
  - 分類 key
  - 狀態
  - 排序
- 多語內容區
  - 保留 EN / RU / ES 分頁
  - 文案改為繁體中文
  - ES 顯示為預留語言
- 產品圖片區
  - 提供「上傳圖片」按鈕
  - 透過既有 `/api/admin/media/upload` 上傳
  - 上傳成功後，立即把新圖片加入當前產品圖庫
  - 顯示目前縮圖預覽
  - 顯示目前產品圖庫預覽
  - 每張圖片可設為縮圖
  - 每張圖片可自圖庫移除

## 資料流

### 圖片上傳

1. 在 `ProductEditor` 中選擇圖片檔案。
2. 呼叫 `/api/admin/media/upload`。
3. 後端完成 WebP 轉換與 R2 上傳。
4. 回傳 `media` 記錄後，前端立即更新本地 `gallery` 狀態。
5. 使用者可將新圖直接設為縮圖。

### 產品儲存

- 儲存時仍呼叫既有 products API。
- `thumbnailMediaId` 與 `gallery` 由 `ProductEditor` 的狀態統一送出。

## 本輪不包含

- 拖曳排序
- 批次圖片上傳
- 圖片裁切
- 媒體選取彈窗
- 首頁內容編輯器表單化

## 驗證標準

- `/admin`、`/admin/products`、`/admin/products/[id]` 介面為繁體中文。
- admin 導覽為左側側欄。
- 在產品編輯頁可直接上傳圖片。
- 上傳後圖片會即時顯示在產品圖庫預覽。
- 可指定任一圖片為產品縮圖。
- 儲存後重新整理仍能正確顯示產品圖片與縮圖。
